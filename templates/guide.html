🛑 Stop-Loss
{% if user.level == 'beginner' %}

Prix en dessous duquel vendre pour limiter les pertes. Calculé automatiquement à partir de la volatilité récente.

{% elif user.level == 'intermediate' %}

Stop = Clôture − 2 × ATR(14). L'ATR mesure l'amplitude moyenne des mouvements sur 14 jours.

{% else %}

Stop_Loss = close − 2 × ATR(14). Méthode Wilder. Déclenche une alerte email si le prix franchit le seuil en intraday (vérification toutes les 15 min).

{% endif %}
💥 Volatilité %
{% if user.level == 'beginner' %}

Mesure l'agitation du titre. Haute volatilité = risque plus élevé mais aussi opportunités plus grandes.

{% else %}

Écart-type annualisé des rendements journaliers sur 20 jours : std(returns_20j) × √252 × 100. Au-delà de 40 % = titre spéculatif.

{% endif %}
{# ── Indicateurs techniques ── #}
Indicateurs techniques
📊 RSI 0–100
{% if user.level == 'beginner' %}

Indique si l'action est trop achetée ou trop vendue. < 30 = survente (rebond possible si confirmé par le MACD), > 70 = surachat (risque de repli).

{% elif user.level == 'intermediate' %}

RSI 14 périodes (Wilder). Zone neutre 40–60. Score +12 pts si RSI 50–70. RSI <30 donne +8 pts uniquement si le MACD confirme un rebond (évite les couteaux qui tombent).

{% else %}

Relative Strength Index, fenêtre 14. RSI = 100 − 100/(1+RS). Score : RSI 50–70 = +12 pts · RSI <30 + MACD_hist >0 = +8 pts · RSI <30 seul = 0 pt · RSI >70 = −5 pts. Volume score : ratio ≥1.5 = +10 pts, ≥1.3 = +5 pts.

{% endif %}
📈 MACD
{% if user.level == 'beginner' %}

Détecte les changements de tendance. Une barre verte dans la colonne MACD signifie une dynamique haussière.

{% elif user.level == 'intermediate' %}

MACD(12,26,9). Histogramme positif = momentum haussier. Croisement MACD > Signal = signal d'achat. +15 pts si histogramme > 0.

{% else %}

EMA12 − EMA26 = ligne MACD · Signal = EMA9(MACD) · Histogramme = MACD − Signal. Score : hist > 0 = +15 pts, croisement haussier récent = bonus.

{% endif %}
🎯 Bollinger %B 0–1
{% if user.level == 'beginner' %}

Position du prix dans ses bandes de Bollinger. < 0.2 = prix bas (potentiel rebond), > 0.8 = prix haut.

{% elif user.level == 'intermediate' %}

%B = (Prix − Bande basse) / (Bande haute − Bande basse). Fenêtre 20, 2 écarts-types. 0 = touche la bande basse, 1 = touche la bande haute.

{% else %}

%B = (close − BB_low) / (BB_high − BB_low). BB(20, 2σ). Combiné avec RSI <30 = signal de rebond fort. Score : %B <0.2 = +5 pts.

{% endif %}
📏 ATR% %
{% if user.level == 'beginner' %}

Amplitude journalière moyenne du titre en %. Un ATR% de 2 % signifie que le titre bouge en moyenne de 2 % par jour.

{% else %}

ATR(14) / Clôture × 100. Mesure la volatilité en termes relatifs. Utile pour calibrer la taille de position : plus l'ATR% est élevé, plus le stop est large.

{% endif %}
🤖 ML Prob. %
{% if user.level == 'beginner' %}

Probabilité calculée par intelligence artificielle que l'action monte de +2 % dans les 10 prochains jours.

{% elif user.level == 'intermediate' %}

Random Forest entraîné sur 100k+ observations. Features : RSI, MACD, Bollinger, Volume, ATR, OBV, Ichimoku. Précision ~66.8 %.

{% else %}

RandomForestClassifier, 200 arbres. Label : close[+10] > close × 1.02. Split chronologique 80/20. Features normalisées (StandardScaler). Boost score : prob >0.65 = +15 pts, >0.55 = +8 pts.

{% endif %}
{# ── Fondamentaux ── #}
Fondamentaux — Valorisation
{# P/E #}
💰 P/E ratio

Définition : Prix / Bénéfice par action. Combien de fois les investisseurs paient le bénéfice annuel.

Interpréter : ≤ 15 = bon marché · 16–25 = raisonnable · > 35 = cher. Un PE négatif = l'entreprise est déficitaire.

Piège : Un PE bas peut cacher une entreprise en déclin. Toujours croiser avec la croissance du CA et le ROE.

{% if user.level == 'expert' %}

Forward PE prioritaire sur trailing. Score : <10=30pts · <15=25pts · <25=20pts · <35=13pts · sinon 5pts.

{% endif %}
{# PEG #}
🚀 PEG ratio

Définition : PE ÷ Taux de croissance des bénéfices. Le PE corrigé de la croissance — l'indicateur de valeur le plus complet.

Interpréter : < 1 = sous-évalué par rapport à sa croissance · 1–2 = juste prix · > 2 = surévalué.

Pourquoi c'est mieux que le PE seul : Apple à PE 30 avec croissance 15 % (PEG 2.0) est moins intéressant qu'une action à PE 20 avec croissance 25 % (PEG 0.8).

Piège : Inutile si la croissance est négative ou nulle. Ne fonctionne pas pour les entreprises matures (utilities, banques).

{# EV/EBIT #}
🏭 EV/EBIT ratio

Définition : Valeur d'entreprise (capitalisation + dettes − trésorerie) divisée par le résultat opérationnel. Mesure combien l'acheteur paye pour chaque euro de profit opérationnel.

Interpréter : < 10 = bon marché · 10–20 = raisonnable · > 25 = cher.

Avantage vs PE : Intègre la dette — une entreprise endettée peut avoir un PE bas mais un EV/EBIT élevé, révélant le vrai coût de l'acquisition.

Piège : EBIT peut être distordu par des éléments non récurrents (cession d'actifs, provisions). Vérifier sur plusieurs années.

{# EV/EBITDA #}
⚙️ EV/EBITDA ratio

Définition : Valeur d'entreprise ÷ EBITDA (bénéfice avant intérêts, impôts, dépréciations et amortissements). L'indicateur préféré des analystes M&A.

Interpréter : < 8 = potentiellement sous-évalué · 8–15 = marché · > 20 = prime de croissance élevée.

Pourquoi l'utiliser : Élimine l'effet comptable des amortissements (très utile pour comparer des entreprises avec des investissements différents). Permet des comparaisons sectorielles.

Piège : L'EBITDA ignore les investissements nécessaires au maintien de l'activité (capex). Une entreprise capital-intensive (usines, mines) peut sembler peu chère en EV/EBITDA mais être chère réellement.

{# FCF #}
💵 FCF cash

Définition : Free Cash Flow = trésorerie générée après tous les investissements. L'argent réellement disponible pour payer des dividendes, racheter des actions ou rembourser des dettes.

Interpréter : Positif et croissant = entreprise saine · Négatif = consomme du cash (normal pour une startup, dangereux pour une entreprise mature).

Pourquoi c'est le meilleur indicateur : Contrairement au bénéfice comptable, le FCF est très difficile à manipuler. Warren Buffett l'utilise en priorité.

Piège : Un FCF négatif peut être volontaire (forte phase d'investissement). Amazon a eu des FCF négatifs pendant 10 ans avant d'exploser.

Affiché en milliards (b) ou millions (m).

Fondamentaux — Rentabilité & Santé
{# P/B #}
📐 P/B ratio

Définition : Prix / Valeur comptable par action. Compare ce que le marché paye à ce que l'entreprise vaut sur le papier (actifs − dettes).

Interpréter : < 1 = décote sur actif net · 1–3 = normal · > 8 = prime très élevée.

Piège : Non pertinent pour les entreprises de services ou tech (actifs immatériels non comptabilisés). Apple a un P/B de 50 — pas parce qu'elle est chère, mais parce que ses marques et brevets n'apparaissent pas au bilan.

{% if user.level == 'expert' %}

Score : <1=20pts · <2=17pts · <4=12pts · <8=6pts · sinon 3pts.

{% endif %}
{# ROE #}
🏆 ROE% %

Définition : Return on Equity = Bénéfice net ÷ Capitaux propres. Mesure l'efficacité avec laquelle l'entreprise utilise l'argent de ses actionnaires.

Interpréter : > 20 % = excellent · 10–20 % = bien · < 5 % = médiocre. LVMH : ~25 %, Apple : ~160 % (via rachats massifs).

Piège : Un ROE très élevé peut être artificiel si l'entreprise est très endettée (la dette réduit les capitaux propres). Toujours vérifier le D/E : ROE élevé + D/E élevé = risque financier.

{% if user.level == 'expert' %}

Score : >25%=25pts · >15%=20pts · >8%=14pts · >0=7pts · négatif=0.

{% endif %}
{# D/E #}
🏦 D/E ratio

Définition : Dettes totales ÷ Capitaux propres. Mesure le niveau d'endettement. Un D/E de 1.5 = l'entreprise a 1.5€ de dettes pour 1€ de fonds propres.

Interpréter : < 0.5 = peu endetté · 0.5–1.5 = normal · > 3 = très endetté.

Piège : Les seuils varient fortement par secteur. Les utilities (EDF, Veolia) et les banques sont structurellement très endettées — comparer toujours au secteur, pas à un seuil absolu.

{% if user.level == 'expert' %}

yfinance retourne ×100 → affiché ÷100. Score : <30=15pts · <80=12pts · <150=9pts · <300=4pts · sinon 0.

{% endif %}
{# Croissance #}
📈 Croiss. CA %/an

Définition : Variation du chiffre d'affaires d'une année sur l'autre. Indique si l'entreprise gagne de nouveaux clients et développe son marché.

Interpréter : > 15 % = forte croissance · 5–15 % = bonne · 0–5 % = stagnation · < 0 % = déclin.

Piège : Croissance du CA ≠ rentabilité. Une entreprise peut tripler son CA en perdant de l'argent. Toujours croiser avec le FCF et le ROE.

{% if user.level == 'expert' %}

Score : >20%=10pts · >10%=8pts · >3%=5pts · >0=3pts · négatif=0.

{% endif %}
{# ── Modèle IA ── #}
Modèle IA (Random Forest)
🤖 Ce que le modèle prédit
{% if user.level == 'beginner' %}

Chaque action est analysée par un algorithme d'intelligence artificielle. Il a appris sur des milliers de situations passées à reconnaître les configurations qui ont précédé une hausse. Il donne une probabilité (0–100 %) que l'action monte de +2 % dans les 10 prochains jours de bourse.

Cette probabilité est utilisée pour ajuster le score final : elle peut l'augmenter ou le diminuer jusqu'à 15 points.

{% elif user.level == 'intermediate' %}

Le modèle est un Random Forest (forêt d'arbres de décision). Il apprend à prédire si close[J+10] > close[J] × 1.02 (hausse > 2 % en 10 jours). Entraîné sur +100 000 observations, split chronologique 80/20, précision ~66–68 %.

Features : RSI, MACD, Bollinger %B, Volume ratio, ATR%, OBV, EMA50, SMA50/200, Ichimoku, ADX, pente SMA200, ATR rang percentile, BB Z-score + régimes de marché.

{% else %}

Label : (close.shift(-10) / close - 1) > 0.02 → binaire 0/1.
Modèle : RandomForestClassifier (200 arbres, max_depth=10, min_samples_leaf=20).
Split : chronologique 80/20 — aucun shuffle (évite le data leakage).
Normalisation : StandardScaler sur X_train, appliqué à X_test.
Boost score : prob ≥ 0.70 = +15 pts · ≥ 0.60 = +8 pts · ≥ 0.50 = +3 pts · ≥ 0.40 = 0 pt · < 0.40 = −15 pts.

{% endif %}
🌍 Régimes de marché
{% if user.level == 'beginner' %}

Le modèle détecte dans quel état se trouve le marché avant d'interpréter les signaux. Un signal haussier a plus de valeur dans un marché en tendance que dans un marché qui oscille sans direction.

  • Marché en tendance (ADX > 25) : les signaux sont amplifiés (+3 pts)
  • Marché en range (ADX < 20) : les signaux de tendance sont atténués (−5 pts)
  • Tendance baissière (SMA200 décroissante) : biais baissier appliqué (−5 pts)
{% elif user.level == 'intermediate' %}

3 indicateurs de régime intégrés dans le score :

  • ADX (Average Directional Index) : mesure la force d'une tendance (pas sa direction). ADX > 25 = tendance, < 20 = range.
  • Pente SMA200 : si la moyenne 200 jours monte → marché structurellement haussier (+3 pts). Si elle descend → biais baissier (−5 pts).
  • ATR Percentile Rank : rang de la volatilité actuelle sur 50 jours. > 70 = volatilité anormalement élevée → utilisé comme feature ML.
{% else %}

ADX(14) via ta.trend.ADXIndicator — 3 niveaux : <20 (range, −5 pts) / 20–30 (neutre) / >30 (tendance forte, +3 pts).
SMA200_slope = (SMA200 − SMA200.shift(10)) / SMA200.shift(10) × 100. slope < 0 → −5 pts.
ATR_pct_rank = ATR_pct.rolling(50).rank(pct=True) × 100. Percentile 0–100.
BB_zscore = (close − BB_middle) / rolling_std(20). Distance normalisée à la BB.
4 régimes backtestés : trend_bull / trend_bear / range_bull / range_bear.

{% endif %}

📏 ADX 0–100
{% if user.level == 'beginner' %}

Mesure la force d'une tendance (pas sa direction). ADX > 25 = tendance forte, ADX < 20 = marché sans direction claire.

{% else %}

Average Directional Index, fenêtre 14. Score : <20 = −5 pts (range) · 20–30 = neutre · >30 = +3 pts (trend fort). Utilisé aussi comme feature ML.

{% endif %}
📉 SMA200 slope %
{% if user.level == 'beginner' %}

Indique si la tendance de long terme monte ou descend. Positif = haussier, négatif = baissier.

{% else %}

Variation de la SMA200 sur 10 jours : (SMA200 − SMA200.shift(10)) / SMA200.shift(10) × 100. Slope < 0 → régime baissier (−5 pts au score).

{% endif %}
🎯 ATR rank 0–100
{% if user.level == 'beginner' %}

Indique si la volatilité actuelle est haute ou basse par rapport aux 50 derniers jours. 100 = volatilité au plus haut.

{% else %}

Percentile rank de l'ATR% sur 50 jours : ATR_pct.rolling(50).rank(pct=True) × 100. >70 = volatilité anormalement élevée (feature ML : regime_vol_high).

{% endif %}
🔢 BB Z-score
{% if user.level == 'beginner' %}

Distance du prix à sa moyenne Bollinger, exprimée en nombre d'écarts-types. > +2 = surachat, < −2 = survente.

{% else %}

BB_zscore = (close − BB_middle) / rolling_std(20). Complémentaire au %B : mesure l'écart absolu (non normalisé 0–1). Vert si < −1 (bas), rouge si > +1 (haut).

{% endif %}
